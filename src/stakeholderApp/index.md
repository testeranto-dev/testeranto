stakeholderApp is an html report generated by the server. It takes the form of a static website which fetches files and which is given a some precomputed data inf the from of graph data (as in graph theory) . The user can tweak the final stakeholder react component to achieve their desired result. we lean on our custom visualization library. 

The stakeholder app allows you to view the application without any implementation details. It shows the results from the perspective "features first". That is, the stakeholder app does not need to reference the source code, logs, or even the results of each test suite. It only shows the specified feature and the pass/fail of the specific, individual tests.

## definitions

"features"  - these are markdown files. Test suits are tagged with these files as paths. 

"feature first" -  The app is designed from the perspective of features. The VS code extension has elements that allow you to view "runtime first" and "process first" views. Stakeholder provides the 3rd view "feature first"

"Test suite" - a group of tests

"test" - a test within a larger suite

## how it works

We have a file `src/stakeholderApp/index.tsx` which we copy to the user's `testeranto` folder during the initialization of the project . The user can edit this file to taste, and during the server start up, this file is bundled (along with the user's changes) by `src/server/serverClasses/Server_Docker/StakeholderAppBundler.ts`. `src/server/serverClasses/utils/embedConfigInHtml.ts` is used to regenerate `testeranto/reports/index.html` dynamically. It computes a graph network and encodes it into the html. Finally, that html loads the compiled stake holder `testeranto/reports/index.js` and the stakeholder app can load.
